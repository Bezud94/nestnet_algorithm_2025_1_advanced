## 2457 - 공주님의 정원

N개의 꽃으로 조건을 만족시키기:
3월 1일부터 11월 30일까지 매일 한 개 이상의 꽃이 피어있도록 하기
가능한 경우의 수 중 정원에 심는 꽃들의 수를 최소로 하기

N <= 100,000 이라서 단순 탐색은 불가능하다.
아~ 이거 생겨먹은 게 딱 그리디인데...?

1. 낙화일이 늦는 순(=내림차순)으로 정렬
2. 위에서 내려오면서 발화일이 3월 1일 이하인 꽃 f1 찾기
3. 다시 위에서 내려오면서 발화일이 f1의 종료일 이하인 꽃 f2 찾기
4. fn의 종료일이 11월 30일 초과일 때까지 반복

낙화일 기준으로 정렬하니 시간 초과, 하지만 접근 방법 자체는 맞는 것 같다

1. 개화일 빠른 순으로 정렬
2. 개화일이 3월 1일 이하인 동안 쭉 탐색 (index)
3. 탐색하면서 가장 발화일이 느린 꽃을 저장해 둔다
4. 탐색이 끝나면 그 꽃을 기준으로 다시 탐색
5. 이 때 index를 연속해서 쓰면 반복할 필요가 없겠다!

## 13460 - 구슬 탈출 2

보드를 기울여 빨간 구슬을 구멍으로 빼내야 한다. 단, 파란 구슬은 빠지면 안 된다.

빨간 구슬을 빼내는 최소 횟수 구하기 / 10번 안에 못 빼면 -1 출력
4^10 = 1,048,576

한 쪽으로 기울이고 그 쪽으로 다시 기울이거나 바로 반대쪽으로 기울이는 건 의미가 없음
따라서 두 번째 이동부터는 유효한 경우의 수가 2개
4 * 2^9 = 2,048

입력받을 때 구슬의 좌표를 따로 저장해두자.
기울일 때는 더 가까운 구슬부터 이동 (구슬이 겹치지 않게 하기 위해서)
예를 들어, 위로 기울였다면 더 위에 있는 구슬부터 이동

어차피 10번만 할 거니까 굳이 이미 왔던 모양인지 확인할 필요는 없을 것 같다.
